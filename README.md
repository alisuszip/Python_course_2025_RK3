# Python_course_2025_RK3
Комплексный дашборд мониторинга ИТ-инфраструктуры и анализа логов

Веб-приложение на Streamlit для мониторинга состояния серверов и анализа логов веб-приложений в реальном времени.

Функционал

Модуль 1: Системные метрики

· Загрузка CSV с метриками серверов через интерфейс
· KPI-карточки с цветовой индикацией состояния (healthy/warning/critical)
· Динамические графики CPU, RAM, Disk, Network с выбором временного диапазона
· Heatmap загрузки CPU по времени и серверам
· Фильтрация по серверам, датам и пороговым значениям

Модуль 2: Анализ логов

· Загрузка CSV с логами веб-приложений
· Статистика HTTP-статусов: гистограмма и распределение (2xx, 4xx, 5xx)
· Топ-10 самых запрашиваемых эндпоинтов и самых медленных
· Поиск по сообщениям и фильтрация по уровням логирования
· Экспорт отфильтрованных данных в CSV

Модуль 3: Интеграционная аналитика

· Корреляционный анализ связи между скачками CPU и ростом ошибок
· Выявление инцидентов: совпадение критических метрик и пиков ошибок
· Автоматические рекомендации по оптимизации
· Графики корреляции с линией тренда

Технологический стек

· Python 3.8+
· Streamlit - веб-фреймворк
· Pandas - обработка данных
· Plotly - интерактивная визуализация
· NumPy - математические вычисления

Структура входных данных

1. Файл метрик (server_metrics.csv)

Формат: timestamp,server_name,cpu_percent,memory_percent,disk_usage_percent,network_in_mbps,network_out_mbps,disk_io_read,disk_io_write,status

Пример:

```
timestamp,server_name,cpu_percent,memory_percent,disk_usage_percent,network_in_mbps,network_out_mbps,disk_io_read,disk_io_write,status
2025-01-15 08:00:00,web-server-01,45.2,67.8,82.1,125.5,89.3,450,320,healthy
```

2. Файл логов (web_app_logs.csv)

Формат: timestamp,level,server_name,client_ip,http_method,endpoint,http_status,response_time_ms,user_agent,message

Пример:

```
timestamp,level,server_name,client_ip,http_method,endpoint,http_status,response_time_ms,user_agent,message
2025-01-15 08:00:01,INFO,web-server-01,192.168.1.100,GET,/api/users,200,45,"Mozilla/5.0...","User list retrieved"
```

Установка и запуск

1. Клонирование репозитория

```bash
git clone <репозиторий>
cd <папка-проекта>
```

2. Установка зависимостей

```bash
pip install streamlit pandas plotly numpy
```

3. Запуск приложения

```bash
streamlit run app.py
```

Приложение будет доступно по адресу: http://localhost:8501

Использование

Загрузка данных

1. Используйте боковую панель для загрузки CSV-файлов
2. Или сгенерируйте тестовые данные с помощью кнопок "Сгенерировать пример"

Навигация

· Системные метрики - мониторинг ресурсов серверов
· Анализ логов - анализ производительности веб-приложений
· Интеграционная аналитика - корреляция метрик и ошибок
· О проекте - описание системы

Генерация тестовых данных

Приложение включает функции генерации реалистичных тестовых данных:

· 4 сервера с различными характеристиками
· 200 записей логов с реалистичными HTTP-статусами
· Временные метки с интервалом 5 минут (метрики) и 30 секунд (логи)

Особенности реализации

Архитектура

· Классовая архитектура с четким разделением модулей
· Обработка ошибок при загрузке файлов
· Кэширование данных для повышения производительности

Визуализация

· Интерактивные графики Plotly с возможностью зума и экспорта
· Цветовая индикация состояния в реальном времени
· Адаптивный дизайн для различных экранов

Аналитика

· Автоматическое определение инцидентов
· Корреляционный анализ с расчетом коэффициента Пирсона
· Интеллектуальные рекомендации на основе данных

· 4 ГБ оперативной памяти (рекомендуется)
